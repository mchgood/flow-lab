# 开发规范（项目约定）

## parser 结构

1. `parser` 使用分包结构：
   - `parser.model`
   - `parser.impl`
   - `parser.exception`
2. 不允许将所有解析相关类直接平铺在 `parser` 根包。

## 实体模型

1. 实体类使用 Lombok 生成样板代码。
2. 类注释和字段注释为必填项。
3. 字段命名与技术方案和类图保持一致。

## 枚举定义

1. 枚举需包含 `code`、`desc` 两个字段。
2. `desc` 用于描述该枚举项语义。

## 引擎分包

1. `ProcessEngine` 相关核心接口与实现不与表达式引擎混放。
2. 表达式相关类放在独立包：
   - `engine.expression`
   - `engine.expression.impl`
3. 节点执行逻辑必须通过 `NodeExecutor` 抽象拆分，不允许全部堆叠在 `DefaultProcessEngine`。
4. 节点间共享数据统一通过 `DataBus`，禁止节点间硬编码依赖调用。
