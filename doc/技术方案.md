# 轻量级 Mermaid 流程引擎设计方案

## （Spring 集成版 v6）

---

# 一、设计目标

基于 Mermaid Flowchart DSL，实现一套可嵌入式单机流程执行引擎，具备：

## 结构能力

* 排他网关（XOR）
* 并行网关（AND）
* 包容网关（OR）
* 子流程嵌套

## 运行能力

* SpEL 条件路由
* 并发控制与 Scope 汇聚
* 节点级能力增强（超时 / 重试 / 异步）
* Scope 级能力增强（整体超时 / 取消策略）
* 全局 DataBus 节点间通信

> 注：`EVENT` 事件网关能力已从当前实现中移除。

## Spring 集成能力

* 每个任务节点映射为 Spring Bean
* 支持节点生命周期拦截器
* 支持流程生命周期拦截器
* 支持按流程 / 按节点精准拦截

---

# 二、总体架构

```
Mermaid DSL
      ↓
Parser
      ↓
ProcessDefinition
      ↓
ProcessEngine (接口)
      ↓
DefaultProcessEngine (单线程调度器实现)
      ↓
ProcessInstance (Runtime)
```

分层结构：

1. 定义层（ProcessDefinition）
2. 控制层（Execution / ScopeRuntime）
3. 行为层（NodeBehavior / ScopeBehavior）
4. 任务层（Spring Bean）
5. 拦截层（NodeInterceptor / ProcessInterceptor）

## 2.1 引擎包结构约定

* 核心流程接口：`com.gaibu.flowlab.engine.ProcessEngine`
* 默认实现：`com.gaibu.flowlab.engine.impl.DefaultProcessEngine`
* 定义容器接口：`com.gaibu.flowlab.engine.definition.ProcessDefinitionRegistry`
* 内存容器实现：`com.gaibu.flowlab.engine.definition.impl.InMemoryProcessDefinitionRegistry`
* 表达式接口：`com.gaibu.flowlab.engine.expression.ExpressionEngine`
* SpEL 实现：`com.gaibu.flowlab.engine.expression.impl.SpelExpressionEngine`

# 三、Spring 任务模型

## 3.1 任务接口

```java
public interface FlowTask {

    void execute(TaskContext context) throws Exception;
}
```

约束：

* 任务只负责业务
* 不允许控制流程走向
* 不允许直接修改 Execution 状态

---

## 3.2 节点与 Bean 映射规则

推荐：

* Node.id = Spring Bean 名称

示例：

```mermaid
S --> approveTask
```

```java
@Component("approveTask")
public class ApproveTask implements FlowTask {
    @Override
    public void execute(TaskContext context) {
        // 业务逻辑
    }
}
```

引擎执行时：

```java
FlowTask task = applicationContext.getBean(node.getId(), FlowTask.class);
```

# 四、执行引擎模型

## 4.1 单线程调度原则（强制）

所有状态变更必须在单一调度线程完成。

模型：

```
调度线程
    ├── 状态修改
    ├── Scope 判定
    ├── 网关路由
    └── 调用拦截器

任务线程池（可选）
    └── 执行 FlowTask
            ↓
       回调调度线程
```

禁止：

* 多线程直接修改 ProcessInstance
* 任务线程修改 Execution 状态

---

## 4.2 统一执行入口

```java
void continueExecution(Execution execution) {

    Node node = getNode(execution.getCurrentNodeId());

    executeNodeWithInterceptors(node, execution);
}
```

## 4.3 引擎接口抽象

```java
public interface ProcessEngine {

    ProcessInstance start(
        String processDefinitionId,
        Map<String, Object> initialVariables
    );
}
```

默认实现：`DefaultProcessEngine`。

运行时外部驱动能力由 `DefaultProcessEngine` 直接提供：

```java
public class DefaultProcessEngine {

    public boolean triggerEvent(
        ProcessInstance instance,
        String eventType,
        String eventKey
    );

    public void onScopeTimeout(
        ProcessInstance instance,
        String scopeId
    );
}
```

## 4.4 节点执行器解耦（NodeExecutor）

为避免 `DefaultProcessEngine` 臃肿，节点执行逻辑按类型拆分为独立执行器：

* `GenericNodeExecutor`：START/TASK/SUB_PROCESS/END
* `ExclusiveGatewayExecutor`：XOR
* `ParallelGatewayExecutor`：AND
* `InclusiveGatewayExecutor`：OR
* `EventGatewayExecutor`：EVENT

调度流程：

1. `DefaultProcessEngine` 仅负责调度与上下文组装
2. 按 `Node` 类型选择对应 `NodeExecutor`
3. 通过 `EngineOperations` 访问引擎内部操作（创建子 execution、scope、路由推进等）

## 4.5 全局数据总线（DataBus）

每个 `ProcessInstance` 持有一个全局 `DataBus`：

* 任意节点可写入：`put(key, value)`
* 任意后续节点可读取：`get(key)`
* 生命周期与 `ProcessInstance` 一致

初始化变量会在 `start(...)` 时同步写入 `DataBus`，便于节点直接读取启动参数。

# 五、节点执行流程（完整生命周期）

节点执行顺序严格定义：

```
NodeInterceptor.before
        ↓
执行 FlowTask
        ↓
NodeInterceptor.onSuccess / onFailure
        ↓
状态更新
        ↓
后续路由
```

# 六、节点拦截器设计

## 6.1 接口定义

```java
public interface NodeInterceptor {

    boolean supports(ProcessDefinition definition, Node node);

    void before(NodeExecutionContext context);

    void onSuccess(NodeExecutionContext context);

    void onFailure(NodeExecutionContext context, Exception e);
}
```

---

## 6.2 拦截匹配规则

允许按以下维度匹配：

* 流程 ID
* 节点 ID
* 节点 metadata
* 节点类型

示例：

```java
@Override
public boolean supports(ProcessDefinition def, Node node) {
    return def.getId().equals("loanProcess")
           && node.getId().equals("approveTask");
}
```

---

## 6.3 拦截器约束

拦截器禁止：

* 修改 execution.status
* 修改 scope
* 跳转流程

允许：

* 日志
* 审计
* 指标统计
* 事务控制
* 安全校验

实现说明（当前）：

* `DefaultProcessEngine` 已接入节点拦截链，执行顺序为 `before -> node execute -> onSuccess/onFailure`
* 节点拦截匹配通过 `supports(ProcessDefinition, Node)` 判定

# 七、流程级拦截器设计

## 7.1 接口定义

```java
public interface ProcessInterceptor {

    boolean supports(ProcessDefinition definition);

    void beforeStart(ProcessExecutionContext context);

    void onCompleted(ProcessExecutionContext context);

    void onFailed(ProcessExecutionContext context, Exception e);
}
```

---

## 7.2 执行顺序

```
ProcessInterceptor.beforeStart
        ↓
流程执行
        ↓
ProcessInterceptor.onCompleted / onFailed
```

流程级拦截器仅在：

* 实例启动
* 流程结束
* 流程失败

实现说明（当前）：

* `beforeStart` 在 `start(...)` 执行前触发
* `onCompleted` 在流程进入终态且无 ACTIVE/WAITING execution 时触发
* `onFailed` 在执行异常时触发（单实例仅通知一次）

时触发。

# 八、网关执行逻辑

## 8.1 XOR

* 按 DSL 顺序评估
* 命中第一个 true 即停止
* default 必须在最后

---

## 8.2 AND

### Split

* 创建 ScopeRuntime
* 创建多个 Execution
* 绑定 scopeId

### Join

```
scope.executions
    .allMatch(e -> e.status == COMPLETED
                && e.currentNodeId == joinNodeId)
```

---

## 8.3 OR

### Split

* 评估所有条件
* 记录 expectedExecutionIds

### Join

```
arrivedExecutionIds.equals(expectedExecutionIds)
```

必须基于 executionId。

---

## 8.4 EVENT

### Split

* 注册多个 EventSubscription
* Execution 进入 WAITING

### 触发

* CAS(WAITING → ACTIVE)
* 删除同 scope 其他订阅
* 恢复执行

# 九、Scope 与 Node 行为优先级

优先级矩阵：

| 优先级 | 事件            |
| --- | ------------- |
| 1   | Scope timeout |
| 2   | Scope cancel  |
| 3   | Node timeout  |
| 4   | Node retry    |
| 5   | 正常完成          |

Scope 行为永远高于 Node 行为。

---

# 十、SpEL 安全约束

允许：

* 变量访问
* 比较运算
* 逻辑运算
* 集合访问

禁止：

* T()
* new
* 反射
* Runtime
* BeanResolver

必须使用受限 EvaluationContext。

---

# 十一、邻接索引强制要求

ProcessDefinition 必须构建：

```java
Map<String, List<Edge>> outgoingIndex;
Map<String, List<Edge>> incomingIndex;
```

运行期禁止扫描 edges。

---

# 十二、不可违反原则

1. 结构层与行为层分离
2. 调度线程唯一
3. Scope 优先级最高
4. Join 必须基于 executionId
5. 拦截器不可改变流程控制

---

# 十三、架构特性总结

本版本引擎具备：

* Spring 深度集成
* 任务完全托管
* 可插拔拦截体系
* 单线程安全调度
* Scope 精确控制
* 企业级扩展能力

它已经不是“流程图执行器”，而是：

> 一个可编排、可扩展、可嵌入的流程执行容器。
