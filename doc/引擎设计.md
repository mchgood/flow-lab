# 🧠 六、InstructionHandler（状态唯一修改点）

```java
public class InstructionHandler {

    public void apply(Instruction instruction, ExecutionContext ctx) {

        switch (instruction) {

            case MoveInstruction move -> move(ctx, move);
            case ForkInstruction fork -> fork(ctx, fork);
            case JoinInstruction join -> join(ctx, join);
            case WaitInstruction wait -> wait(ctx, wait);
            case CompleteInstruction complete -> complete(ctx);
            case FailInstruction fail -> fail(ctx);
        }
    }
}
```

# 🔀 七、并行模型（Token 级别）

---

## Fork（真正正确的方式）

```java
private void fork(ExecutionContext ctx, ForkInstruction instr) {

    Token current = ctx.token();

    ctx.removeToken(current);

    for (NodeId next : instr.nextNodes()) {
        Token newToken = TokenFactory.create(next, current.getExecution());
        ctx.addToken(newToken);
        scheduler.schedule(newToken);
    }
}
```

}

---

## 🔥 模板（你原来的升级版）

```java
public class NodeExecutionTemplate {

    public Instruction execute(NodeBehavior behavior, ExecutionContext ctx) {

        interceptor.before(ctx);

        try {
            Instruction instr = behavior.handle(ctx);
            interceptor.afterSuccess(ctx, instr);
            return instr;
        } catch (Exception e) {
            interceptor.afterFailure(ctx, e);
            throw e;
        }
    }
}
```
